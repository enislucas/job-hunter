# Corrections inbox

Standing rules and one-off corrections from you, in force until you change them. Every run reads this file FIRST, before anything else. Corrections here override your search preferences and every assumption.

Write in free words: a package you refused and why, a CV line you edited, a style complaint, a values objection or a new ruling, a status update ("submitted to Borealis Logistics", "got an interview at X", "rejected by Y"). The run parses status updates into the journey log and ledger, turns corrections into numbered rules, and moves any values ruling into memory\shaykh-rulings.md. Processed items are deleted from this file (the header stays); nothing is lost, it moves into the logs.

(This file starts empty. Add a line whenever you want to teach the machine something.)
