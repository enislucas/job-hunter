# Brief template (daily)

Files: briefs\YYYY-MM-DD-brief.md plus self-contained html twin (same tokens as dashboard). Sections, fixed order, honest numbers only:

1. TLDR, max 5 lines: what happened, the single hottest item, what needs the candidate.
2. Packages prepared today: table (company, role, one-line why, deadline, link to application page). Each row's why names the evidence fit, not adjectives.
3. Values queue (if the values module is on): open questions, each readable in 30 seconds, template per HALAL-RUBRIC-TEMPLATE.md.
4. Skills delta: any rank moves in SKILLS.md, plus the featured project (why, first three steps).
5. New glossary entries, one line each.
6. LinkedIn suggestion, only if evidence shifted.
7. Interview kit: new insights distilled, outreach drafts waiting for the candidate's send.
8. Corrections applied and rules changed: C-numbers, what changed, one line each.
9. Tips: new proposals with the three choices; parked count.
10. Needs your call: every hard-stop item, each with the safe default that applies if the candidate stays silent (silence never means submit).
11. Honest numbers: prepared today and total, submitted, responses, interviews, acceptance rate, run spend (sub agents, tokens if visible), usage meter reading.
